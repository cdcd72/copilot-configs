# Test

## 目的

執行現有測試、修復失敗案例，或為關鍵邏輯補充測試，以確保程式碼的正確性與強健性。

## 流程

1. 先查看 `package.json` 的 `scripts`，確認可用的測試指令，例如 `test`、`test:unit`、`test:coverage`。
2. 使用 PowerShell 執行找到的測試指令；若命令失敗，記錄錯誤訊息並分析原因。
3. 若測試失敗，優先修正原始程式碼邏輯，而不是單純修改測試讓它通過。
4. 若專案缺乏測試或覆蓋率不足，為核心商業邏輯補充具備原子性的單元測試，避免依賴外部環境。

## 參考指令

```pwsh
Select-String -Path package.json -Pattern '"scripts"' -Context 0,15
```

## 限制

- 若專案沒有可執行的測試腳本，要先說明原因再決定是否補測
- 除非確認測試案例本身有誤，否則不要只改測試來消除失敗

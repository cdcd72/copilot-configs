# Review

## 目的

針對當前工作目錄下的程式碼變更進行嚴格且客觀的技術審查，直接指出問題並提供具體建議。

## 審查標準

1. 可讀性與規範：命名是否精確、邏輯是否易懂、是否符合語言慣用寫法。
2. 安全性：是否存在注入攻擊、資料外洩或錯誤處理不當的風險。
3. 效能與資源：是否有不必要的運算、記憶體洩漏或 N+1 查詢問題。
4. 程式碼異味：是否有重複邏輯、過度複雜的判斷式或過長的函式。

## 建議流程

1. 查看檔案狀態。

   ```pwsh
   git status -s
   ```

2. 閱讀詳細變更。

   ```pwsh
   git diff HEAD
   ```

3. 依嚴重程度排序列出問題，再補上必要的假設與風險。
